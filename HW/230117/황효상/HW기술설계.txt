
1. MQTT를 사용해야하는 이유
    
    
    - 확장성
    
    MQTT 작업 시 적은 전력만 소비되고, IoT 디바이스와의 통신을 지원하기 위한 기능을 갖추고 있어, 여러 개의 디바이스에 연결할 수 있습니다.
    
    - 경량화
    
    경량 메시지 포맷을 사용하고, 80~100 Kb 코드 메모리만 사용하므로 간단한 구현이 가능합니다.

2. Aruino vs Raspberry Pi 4

    아두이노 : adc 컨버터 없이 아날로그 신호 처리 가능

    Raspberry Pi 4 : 메모리, 빠른 성능, 간편한 개발 -> 하지만 ADC 컨버터 필요

    필요 기능 : lcd모니터 출력, 압력, 온습도, 조도 등 6개 센서 필요

    -> Arduino 두개는 lcd모니터 사용으로 핀부족, Raspberry Pi 는 adc컨버터가 많이 필요해서 회로 구성복잡

    결론 : Arduino 1개와 Raspberry Pi 4를 시리얼 통신하여 arduino는 센서데이터만 측정하여 pi로 전송

    Raspberry Pi 4 는 모니터 출력과 센서데이터 수신 및 웹으로 데이터 전송 기능
